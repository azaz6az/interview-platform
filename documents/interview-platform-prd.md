# 面面俱到 — 大学生面试备战平台 PRD

## 1. 项目信息

| 字段 | 值 |
|------|-----|
| Language | 中文 |
| Programming Language | Vite + React 18 + MUI v5 + Tailwind CSS（tw- 前缀） |
| Project Name | interview_prep_platform |
| 原始需求 | 为大数据管理与应用专业、即将找实习的大学生，开发一款面试备战 Web 平台，解决「不知道考什么、没地方练、练了不知道有没有进步」三大痛点 |

## 2. 产品目标

1. **精准定位差距** — 帮助用户快速发现自身能力与目标岗位之间的差距，并提供可执行的改进建议
2. **低门槛实战练习** — 提供零压力的 AI 模拟面试环境，让用户随时随地练习并建立信心
3. **数据驱动进步** — 通过复盘记录和 AI 分析，让用户清晰看到自己的薄弱环节和进步轨迹

## 3. 目标用户

**核心用户画像**：
- 大学生，大数据管理与应用专业
- 有 CS 基础但不精通
- 即将开始实习求职
- 面试经验不足，缺乏系统化的准备方法
- 需要一个工具帮助自己拿到实习 Offer

## 4. 用户故事

### 模块一：JD 简历适配（已有模块集成）

- **US-1.1**：作为一名求职学生，我希望粘贴我的简历和目标岗位 JD 后能一键分析匹配度，这样我就知道哪些关键词缺失、哪些经历需要调整
- **US-1.2**：作为一名求职学生，我希望分析结果能给出具体的修改建议（关键词补充、措辞调整、优先级调整、技能匹配），这样我不用自己猜测怎么改
- **US-1.3**：作为一名求职学生，我希望修改后的简历能一键复制，这样我可以快速更新我的投递版本

### 模块二：AI 模拟面试

- **US-2.1**：作为一名面试新手，我希望选择岗位方向（如数据分析、产品运营等）后开始模拟面试，这样练习内容与我的求职方向匹配
- **US-2.2**：作为一名面试新手，我希望 AI 扮演面试官进行多轮问答（至少 3 轮追问），这样我能体验真实的面试节奏和压力
- **US-2.3**：作为一名面试新手，我希望模拟面试结束后获得评分和具体反馈（回答质量、逻辑性、表达清晰度），这样我知道哪里可以改进
- **US-2.4**：作为一名面试新手，我希望模拟面试的记录自动保存到复盘日记，这样我不用手动重复录入

### 模块三：面试题库

- **US-3.1**：作为一名求职学生，我希望按岗位方向浏览面试题，这样我能针对性地准备
- **US-3.2**：作为一名求职学生，我希望按难度筛选题目（入门/中级/困难），这样我可以循序渐进地练习
- **US-3.3**：作为一名求职学生，我希望搜索特定关键词找到相关题目，这样我能快速定位想练的内容
- **US-3.4**：作为一名求职学生，我希望收藏重要题目，这样我可以建立自己的重点练习集

### 模块四：面试复盘日记

- **US-4.1**：作为一名求职学生，我希望记录每次面试（真实或模拟）的题目、我的回答和感受，这样我不会忘记面试中学到的东西
- **US-4.2**：作为一名求职学生，我希望 AI 能分析我的复盘记录并识别薄弱环节，这样我可以有针对性地补强
- **US-4.3**：作为一名求职学生，我希望看到我的进步趋势（如薄弱项减少、评分提升），这样我有动力继续练习

### 模块五：公司情报卡（P1）

- **US-5.1**：作为一名求职学生，我希望输入公司名后看到业务介绍、面经摘要、薪资范围，这样我能为特定公司做针对性准备

### 模块六：投递追踪看板（P1）

- **US-6.1**：作为一名求职学生，我希望在看板上管理投递状态（待投/已投/面试中/已拒/已录），这样我不会遗漏跟进
- **US-6.2**：作为一名求职学生，我希望在看板卡片上关联 JD 和简历，这样我能快速回顾每家公司的投递材料

## 5. 需求池

### P0 — MVP 必做

| 编号 | 名称 | 描述 | 验收标准 |
|------|------|------|----------|
| P0-01 | JD 简历适配器集成 | 将已有的独立模块集成到平台导航体系，保持原有三栏布局和分析功能 | 1. 在平台导航中可进入 JD 适配页 2. 原有分析功能正常 3. 分析结果可一键复制 4. localStorage 数据持久化正常 |
| P0-02 | 岗位方向选择 | 支持选择模拟面试的岗位方向 | 1. 提供至少 5 个岗位方向（数据分析、产品运营、商业分析、数据开发、管培生）2. 选择后影响出题内容 |
| P0-03 | AI 模拟面试核心流程 | AI 扮演面试官进行多轮问答 | 1. AI 发起首个问题 2. 用户回答后 AI 追问至少 2 轮 3. 面试持续 3-8 轮 4. 有"结束面试"按钮 5. 结束后展示评分和反馈 |
| P0-04 | 模拟面试评分反馈 | 面试结束后给出多维度评分和文字反馈 | 1. 评分维度至少包括：回答质量、逻辑性、表达清晰度 2. 每维度 1-5 分 3. 提供具体改进建议 4. 总评分和各维度评分可视化展示 |
| P0-05 | 面试题库浏览 | 按岗位方向和难度分类展示题目 | 1. 至少 50 道预置题目覆盖 5 个岗位方向 2. 每道题标注难度（入门/中级/困难）3. 支持按岗位和难度筛选 4. 点击题目可查看参考答案 |
| P0-06 | 题库搜索与收藏 | 支持关键词搜索和收藏题目 | 1. 搜索框输入关键词实时过滤 2. 收藏/取消收藏操作 3. 有"我的收藏"筛选入口 4. 收藏数据持久化到 localStorage |
| P0-07 | 复盘日记创建 | 记录每次面试的详细信息 | 1. 可选择面试类型（真实/模拟）2. 必填：日期、面试题目 3. 选填：我的回答、感受评分（1-5）4. 保存到 localStorage |
| P0-08 | 复盘薄弱环节分析 | AI 分析复盘记录，识别薄弱环节 | 1. 汇总复盘记录中的题目和评分 2. 识别得分最低的知识/能力维度 3. 展示 Top 3 薄弱环节 4. 为每个薄弱环节推荐练习题 |

## 6. UI 设计稿描述

### 6.1 整体布局

- **导航模式**：顶部 AppBar + 左侧抽屉式导航（Drawer），适配桌面和移动端
- **主色调**：以 MUI 默认蓝紫为基调，搭配暖橙色为强调色（代表"积极备战"）
- **响应式**：桌面端侧边栏常驻，移动端底部 Tab 栏

### 6.2 页面结构

```
┌─────────────────────────────────────────────┐
│  AppBar: Logo "面面俱到" + 用户头像 + 设置    │
├──────────┬──────────────────────────────────┤
│ Drawer   │  Main Content Area               │
│          │                                  │
│ 🏠 首页  │  (根据选中导航切换)               │
│ 📄 JD适配│                                  │
│ 🎤 模拟面│                                  │
│ 📚 题库  │                                  │
│ 📝 复盘  │                                  │
│ 🏢 情报卡│                                  │
│ 📋 看板  │                                  │
└──────────┴──────────────────────────────────┘
```

### 6.3 首页（Dashboard）

- 顶部问候语 + 最近一场面试倒计时（如有 P2-01）
- 四宫格快捷入口卡片：JD 适配、模拟面试、题库练习、复盘日记
- 底部：最近 3 条复盘记录摘要 + 薄弱环节 Top 3 小组件

### 6.4 JD 简历适配页

- **复用已有模块的三栏布局**：简历输入 | JD 输入 | 分析结果
- 集成到平台后，Header 部分由平台统一管理，原有 Header 组件替换为页面标题
- 底部增加「将分析结果保存到复盘日记」快捷按钮

### 6.5 AI 模拟面试页

- 进入页：岗位方向选择卡片（网格布局，每个方向一张卡片带图标）
- 面试中：对话式 UI
  - 左侧/上方：AI 面试官头像 + 消息气泡
  - 右侧/下方：用户输入区域（多行文本框 + 发送按钮）
  - 顶部进度条显示当前轮次
  - 右上角"结束面试"按钮
- 面试结束：评分结果页
  - 多维度雷达图 + 各维度分数 + 文字反馈
  - 「查看详细复盘」和「再试一次」按钮

### 6.6 面试题库页

- 顶部：搜索栏 + 岗位筛选 Chips + 难度筛选 Chips
- 主体：题目列表（Card 布局）
  - 每张卡片：题目标题、岗位标签、难度标签（颜色区分）、收藏按钮
  - 点击展开：题目详情 + 参考答案（默认折叠）
- 右侧/底部：我的收藏快捷筛选

### 6.7 复盘日记页

- 顶部：新建复盘按钮 + 薄弱环节分析卡片
- 主体：时间线视图（Timeline）
  - 每条记录：日期、面试类型标签（真实/模拟，颜色区分）、公司名、评分星级
  - 点击展开：题目列表、我的回答、感受、AI 分析建议
- 底部：进步趋势图（近期评分折线图）

### 6.8 公司情报卡页（P1）

- 顶部：搜索框
- 主体：公司信息卡片
  - 左侧：公司 Logo + 名称
  - 右侧：Tab 切换（简介 / 面经 / 薪资 / 动态）
  - 每个 Tab 下展示对应信息

### 6.9 投递追踪看板页（P1）

- 五列看板布局：待投 → 已投 → 面试中 → 已拒 → 已录
- 每列内：投递卡片（公司名、岗位、日期）
- 点击卡片：展开详情（关联 JD、简历版本、面试日期）

## 7. 待确认问题

| # | 问题 | 影响范围 | 建议 |
|---|------|----------|------|
| 1 | AI 模拟面试的"AI"在 MVP 阶段是纯前端模拟还是需要对接真实 LLM API？ | P0-02, P0-03, P0-04 | MVP 阶段用前端预设题库 + 规则匹配模拟 AI 行为，预留 `generateQuestion(role, history)` 和 `evaluateAnswer(question, answer)` 异步接口 |
| 2 | 面试题库的初始 50 道题内容来源？是手动编写还是抓取？ | P0-05 | 建议手动编写高质量题目，覆盖大数据管理与应用专业常见面试场景，避免版权问题 |
| 3 | 复盘日记的薄弱环节分析维度如何定义？ | P0-08 | 建议定义 5-6 个能力维度（如：专业知识、项目经验、沟通表达、逻辑思维、技术深度、行业认知），每道题关联 1-2 个维度 |
| 4 | JD 适配器模块集成时是否需要重构目录结构？ | P0-01 | 建议将现有 `jd-resume-adapter` 的 `src/` 下代码迁移到新项目的 `src/modules/jd-adapter/` 目录，保持模块边界清晰 |
| 5 | 平台是否需要用户登录体系？ | 全局 | MVP 阶段不需要，所有数据存 localStorage；预留用户 ID 字段，后续可扩展 |
| 6 | 公司情报卡的数据来源？ | P1-01 | 前端 mock 预置 10-20 家热门公司数据，预留 `fetchCompanyInfo(name)` API 接口 |
| 7 | 移动端底部 Tab 栏最多放几个导航项？ | 全局 | 建议 5 个：首页、JD适配、模拟面试、题库、复盘；P1 功能放入"更多"菜单 |

## 8. 技术约束

- **纯前端 Web 应用**：Vite + React 18 + MUI v5 + Tailwind CSS（tw- 前缀）
- **无后端**：AI 能力用前端预设逻辑模拟，所有 AI 相关功能预留 async API 接口
- **数据持久化**：localStorage，按模块划分 key（如 `ip_resume`、`ip_interview_records`、`ip_favorites`）
- **已有模块集成**：JD 简历适配器代码迁入 `src/modules/jd-adapter/`，保持组件和引擎代码不变
